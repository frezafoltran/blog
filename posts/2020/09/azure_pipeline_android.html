How to Deploy an Android app on Azure

Building, signing and deploying an Android app using Azure Pipelines

Posted by Foltran on September 15, 2020

Initial Setup

Since an Android app does not require a specific VM to be built on, we can just follow the steps in the iOS guide.

Handle Dependencies

To install dependencies of the Android app, simply run a yarn install task.

Build the app

On Android, the configurations for building the app are in the gradle file. The task below uses the config in the gradleWrapperFile to build the app.

Sign the app

We can generate an authentication key to sign the Android app using Visual Studio. When we have a .keystore file, signing the app is done by using the AndroidSigning task on Azure.

Publish build Artifact

The part for publishing the build artifact is a boilerplate code and is included in the full version of the pipeline below. Further, you can check out TODO to learn how to create a release pipeline to push the .apk to AppCenter.
